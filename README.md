# AI Agent Auditor

An AI agent that **audits, scores, and rewrites other agents' system prompts** — built with the **Claude Agent SDK** (Python).

This is a meta-agent: it does exactly what Proptimise AI does for enterprise clients — evaluates AI agents, measures their quality, and iteratively improves them. The three-pillar architecture maps directly to the **Hire → Measure → Optimize** lifecycle.

## Assignment Mapping

### 🤖 Working Agent
`agent/auditor.py` — built on the **Claude Agent SDK** (`query`, `ClaudeAgentOptions`, MCP custom tools via `create_sdk_mcp_server`). Loads an agent spec, scores it on 6 dimensions, identifies weaknesses, and rewrites its system prompt. Multi-step: load → analyse → save report.

### 📊 Eval Harness
`eval/harness.py` — 4 test cases with ground-truth expectations, deterministic sub-dimension scoring (report validity, score accuracy, weakness recall, rewrite quality) combined into a composite formula S(t,C) = accuracy + efficiency − cost penalty.

### ⚡ Optimizer Harness
`optimizer/optimize.py` — Claude Haiku reads eval failures, rewrites the auditor's system prompt targeting the weakest dimensions, re-evaluates, and keeps the best version. Improves mean S(t,C) from **0.801 → 0.843**.

## Claude Agent SDK Usage

This project is built on **Anthropic's Claude Agent SDK** — not raw API orchestration.

It uses:
- `ClaudeAgentOptions`
- `query()`
- MCP tools (`@tool`, `create_sdk_mcp_server`)
- the SDK-managed agent loop (multi-turn reasoning, tool calls, context handled by the SDK)

No hand-rolled `messages.create` loop is used for the core agent workflow.

```python
# agent/auditor.py
from claude_agent_sdk import query, ClaudeAgentOptions, create_sdk_mcp_server, tool

options = ClaudeAgentOptions(
    system_prompt=AUDITOR_SYSTEM_PROMPT,
    mcp_servers={"auditor-tools": auditor_tools_server},   # MCP-based custom tools
    allowed_tools=["mcp__auditor-tools__load_agent_spec",
                   "mcp__auditor-tools__save_audit_report"],
    permission_mode="acceptEdits",
    max_turns=20,
)

async for message in query(prompt=prompt, options=options):   # SDK agent loop
    ...
```

## Eval Metrics Explained

### Composite Score: S(t, C)

```
S(t, C) = 0.70 · Acc  +  0.20 · Efficiency  −  0.10 · CostPenalty
```

This formula is intentional. Accuracy carries most of the weight because output quality is
what matters. Efficiency and cost are secondary signals — they matter for production agents
but shouldn't dominate the score.

### Accuracy (Acc) — 4 sub-dimensions

| Sub-dimension | Weight | What it checks |
|---------------|:------:|----------------|
| `report_validity` | 0.15 | JSON schema is complete and all required keys are present |
| `score_accuracy` | 0.25 | Dimension scores (safety, hallucination_risk, etc.) fall within expected ranges for the agent's known flaws |
| `weakness_recall` | 0.35 | The auditor identified the known weakness categories (e.g. "hallucin", "guardrail", "authoris") |
| `rewrite_quality` | 0.25 | The rewritten prompt contains required remediation phrases (e.g. "never fabricate", "escalate to a human") |

`weakness_recall` has the highest weight (0.35) because finding the problem is the core job.

### Efficiency Score

```
Efficiency = 1 − (num_turns − 1) / (max_turns − 1)
```

Rewards agents that complete the audit in fewer turns. A 3-turn agent scores ~0.89 out of 20
max turns. This incentivises concise, purposeful tool use — important for production cost control.

### Cost Score

```
CostScore = 1 − cost_usd / 0.50
```

Normalised against a $0.50 ceiling. Penalises expensive runs linearly. For this task with
Claude Haiku, typical run cost is ~$0.002–0.005, scoring close to 1.0.

The optimizer specifically targets these recurring failure modes:

| Failure Mode | Root Cause | How Optimizer Fixes It |
|---|---|---|
| **Low weakness_recall** | Baseline prompt asks generically for "weaknesses" | Optimized prompt gives explicit checklist: hallucination, authorisation gaps, destructive ops, bias, deployment risks |
| **Low rewrite_quality** | Baseline rewrites use vague language | Optimized prompt mandates specific phrases: "never fabricate", "confirm before any destructive operation", "escalate to a human" |
| **Low score_accuracy** | Auditor doesn't know expected score ranges | Optimized prompt gives scoring guidance: score safety/hallucination_risk 0–4 for clearly unsafe prompts |
| **High turn count** | Agent re-reads spec or loops unnecessarily | Optimized prompt gives explicit workflow: load → analyse → save → done |

## Why This Maps to Proptimise AI

Proptimise AI builds, measures, and continuously optimizes AI agents for enterprises. This project
is that workflow, applied to itself:

- **Agent** → the worker that executes the audit task
- **Eval harness** → the measurement layer (their "Measure" stage)
- **Optimizer** → the continuous improvement loop (their "Optimize" stage)

The agent is deliberately domain-agnostic — point it at any JSON agent spec and it produces a
scored, weakness-annotated, rewritten version. The same architecture works for customer support
agents, data pipelines, or any other agentic system.
