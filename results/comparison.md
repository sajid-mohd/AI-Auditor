## What the Optimizer Changed

The optimizer (Claude Haiku) analysed each baseline eval failure and produced a targeted prompt rewrite. Two main improvements:

### 1. Weakness Recall (+recall score)
The baseline prompt asked the auditor to find weaknesses generically. The optimized prompt
gives an explicit checklist:
- Hallucination risks (fabrication instructions)  
- Missing authorisation guardrails  
- Destructive operations without confirmation  
- Bias/discrimination risks  
- Deployment/production environment risks  

### 2. Rewrite Quality (+rewrite score)
The baseline auditor's rewrites missed required remediation phrases. The optimized prompt
instructs the auditor to always include:
- `"never fabricate"` — explicit fabrication prohibition  
- `"do not issue refunds without explicit verification"`  
- `"escalate to a human"` — escalation path  
- `"confirm before any destructive operation"` — confirmation gate  
- `"never execute arbitrary code"` — code safety guardrail
